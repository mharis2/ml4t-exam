Error Functions and Training
The training objective is to minimize the error function (loss) :
E(w) = (1/2) × Σ [y(x ₙ , w) - t ₙ ]²     (sum of squared errors)
w = weights (parameters to be learned)
y(xₙ, w)  = model prediction for input xₙ
tₙ = true target value
The factor 1/2 is for convenience (derivatives become cleaner)
For linear models, this has a closed-form solution. For deep neural networks, it must be solved
iteratively using gradient descent.
Regularization (Weight Decay / L2 Penalty)
Problem:  High-order models overfit by tuning weights to huge values to perfectly match
training noise.
Solution:  Add a penalty on large weights to the error function:
Ẽ(w) = (1/2) × Σ [y(x ₙ ,w) - t ₙ ]² + (λ/2) × ||w||²
Where ||w||² = w₀² + w₁² + ... + w ₘ ².
🎯 Exam Flag: This is L2 r egular ization (also called Ridge or w eight decay in neural
netw orks). The λ p aramet er contr ols ov erfitting:
- λ too small  → overfitting (effectively no regularization)
- λ too lar ge → underfitting (weights forced too close to zero)
- λ just r ight → controlled complexity, good generalization
The regularizer penalizes large coefficients, keeping them small and smooth — preventing the
wild oscillations seen in unregularized high-order polynomials.
Hyperparameters and Model Selection
Hyperp aramet ers = parameters set befor e training that control model complexity (not learned
from data):
- M = polynomial order (or number of layers / hidden units in a neural network)
- λ = regularization strength
You cannot select hyperparameters by minimizing training error alone — that always leads to
largest M and λ → 0 (maximum overfitting).
Solution: V alidation set + Cr oss-v alidation

Training set → Fit model parameters (weights)
Validation set → Select hyperparameters (M, λ)
Test set → Final unbiased evaluation (touch ONCE)
K-fold cr oss-v alidation:
1. Split data into K equal groups
2. Train on K-1 folds, validate on 1 fold
3. Rotate K times, average scores
4. Best hyperparameter = lowest average validation error

Backpr opagation
Information flows forward through the network (forward pass) to compute predictions. Errors
flow backward (backward pass) to compute gradients.
Forward pass:  inputs → hidden layers → output → loss
Backward pass: loss → gradients flowing back through each layer
Each parameter's gradient tells us: "Which direction should I move this weight to decrease the
loss?" Then: Stochastic Gradient Descent (SGD)  updates weights one mini-batch at a time.

Layer 4 Full objects
For trading: Layer 1 might learn individual candlestick patterns, later layers learn multi-day
regimes. This automatic feature extraction is why deep learning dominates when data is
abundant.

Key insight of the deep learning era: more data + mor e comput e + bigger model = bett er
performance , often following power laws.
From 2012 onward: compute used to train state-of-the-art models grew 10× per year
Large Language Models (GPT, BER T, Claude): same architecture, different domains — one

Transfer Learning: Pre-train on massive general dataset → fine-tune on your specific task.

The Cur se of Dimensionality
As the number of predictors p grows:
- kNN must search in higher-dimensional space
- Neighbors become farther away — the "nearest" points are no longer truly close
- Average over distant neighbors = high bias
- kNN performance degrades rapidly with p even if the true relationship is non-linear

🎯 Exam Flag: kNN is competitiv e for p=1 or p=2 featur es, but linear r egression o ften
wins for p ≥ 4 ev en when the tr ue relationship is non-linear .
Trader lesson: If you have 50 features (volume, momentum, various technical indicators), linear
regression or tree-based methods will typically outperform kNN. kNN works when you have
few, carefully engineered features.
Ridge Regression (L2 Regularization)
Problem:  When the number of predictors p is close to n (or p > n), ordinary least squares (OLS)
has very high variance — small changes in training data → wildly different coefficient estimates.
Ridge solution:  Add an L2 penalty to the loss function:
Minimize: RSS + λ × Σβⱼ²
         = Σ(yᵢ - ŷᵢ)² + λ(β₁² + β₂² + ... + β ₚ ²)
🎯 Exam Flag: λ contr ols the bias-v ariance tradeo ff in Ridge:
- λ = 0 : Ridge = OLS (no regularization)
- λ → ∞ : All β → 0 (null model, max bias, zero variance)
- Optimal λ : chosen via cross-validation
Effect o f Ridge:
- All coefficients shrink towards zero but never reach exactly zer o
- Ridge keeps ALL predictors in the model (no feature selection)
- Reduces variance at the cost of a small increase in bias
- Works best when OLS has high variance (p close to n, or highly correlated predictors)
Key requir ement:  Standardize predictors first (mean=0, std=1), because Ridge penalizes
coefficient magnitude and scale matters.
Lasso (L1 Regularization)
Lasso  (Least Absolute Shrinkage and Selection Operator) uses an L1 penalty:
Minimize: RSS + λ × Σ|βⱼ|
         = Σ(yᵢ - ŷᵢ)² + λ(|β₁| + |β₂| + ... + |β ₚ |)
🎯 Exam Flag: The k ey differ ence fr om Ridge — Lasso sets some coefficients t o EXACTLY
zero (per forms aut omatic featur e selection).
Why does Lasso cr eate zer os?
- Geometrically: the L1 constraint region is a diamond with corners at the axes
- The RSS contours often intersect the L1 diamond at a corner → sets that coefficient to 0
- The L2 circle (Ridge) has no corners → intersection is never exactly on an axis

Effect o f Lasso:
- Sparse models : only a subset of predictors have non-zero coefficients
- Automatic featur e selection : eliminates irrelevant features
- More interpretable than Ridge

The Cor e Idea
Every stock return = two parts:

2. Comp any-specific r eturn (idiosyncratic risk — can diversify away)
CAPM equation:
Return(stock) = Alpha + Beta × Return(market) + ε
This is literally a simple linear regression of stock returns on market returns.
Alpha (α)
🎯 Exam Flag:
Alpha Meaning
α > 0 Stock outperforms the market on a risk-adjusted basis = investing skill
α = 0 Stock performs in line with what market risk predicts
α < 0 Stock underperforms market on risk-adjusted basis

Alpha  = Y-intercept of the regression line (stock return vs. market return scatter)
On average across all stocks, alpha = 0 (because all stocks together = the market). Any stock
with positive alpha is offset by another with negative alpha.
"Seeking alpha " = finding stocks with genuine excess return above what their market risk
would predict. This is the core goal of active management.
Beta (β)
🎯 Exam Flag:
Beta Meaning
β = 1.0 Moves exactly with the market
β > 1.0 More volatile than market (amplified swings) — speculative stocks
β < 1.0 Less volatile than market — stable large companies
β = 0 No correlation to market
β < 0 Moves opposite the market
Beta  = slope of the regression line (stock return vs. market return)
Examples:
- ExxonMobil (X OM): β ≈ 0.77 (large, stable company, slightly less volatile than market)
- Small speculative biotech: β often 1.5–3.0
- Gold ETF: sometimes negative β (safe-haven asset)
"Buying beta " = investing in high-beta stocks for amplified market exposure. Higher expected
return but also higher risk.
Correlation
Correlation coefficient  (−1 to +1) measures how much two assets move together:
- +1.0 : Perfect positive correlation (always move together)
- 0: No relationship
- −1.0 : Perfect negative correlation (always move oppositely)
🎯 Exam Flag: Combining assets with low or negativ e corr elation r educes por tfolio

🎯 Exam Flag: The beta-neutral/mark et-neutral strat egy is a cor e hedge fund appr oach.

- Hedge funds leverage 20-30x to amplify these small alphas into meaningful returns

🎯 Exam Flag: Sur vivorship bias is a major pitfall in b acktesting.
